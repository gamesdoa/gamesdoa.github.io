SpringBoot 使用 jasypt 3.0 配置文件敏感信息加密

发表于 2020-11-19 | 分类于 SpringBoot | | 阅读次数
SpringBoot 使用 jasypt 3.0 配置文件敏感信息加密。

说明

SpringBoot配置文件中资源文件中的内容通常情况下是明文显示，安全性比较低些在 application.properties或application.yml中，MySql登陆密码，Redis登陆密码以及第三方的密钥等等，如何对这些敏感信息进行加密，提高属性配置的安全性呢？

这里可以使用jasypt来处理，该工具是由国外的一位大神编写的springboot下的工具包，用来加密配置文件中的信息。

数据用户名和数据库密码加密为例

  • 引入包

    查看最新版本可以到:https://github.com/ulisesbocchio/jasypt-spring-boot

    <dependency>
    <groupId>com.github.ulisesbocchio</groupId>
    <artifactId>jasypt-spring-boot-starter</artifactId>
    <version>3.0.3</version>
</dependency>

  • 配置加/解的密码

    # jasypt加密的密匙及加密模式
jasypt:
  encryptor:
    iv-generator-classname: org.jasypt.iv.NoIvGenerator
    algorithm: PBEWithMD5AndDES
    password: test_key

  • 测试用例中生成加密后的秘钥

    public static void main(String[] args) {
    StandardPBEStringEncryptor standardPBEStringEncryptor = new StandardPBEStringEncryptor();
    standardPBEStringEncryptor.setAlgorithm("PBEWithMD5AndDES");
    standardPBEStringEncryptor.setPassword("test_key");
    //加密
    String username = standardPBEStringEncryptor.encrypt("root");
    String password = standardPBEStringEncryptor.encrypt("root");
    System.out.println("username:"+username);
    System.out.println("password:"+password);
    //解密
    String username2 = standardPBEStringEncryptor.decrypt(username);
    String password2 = standardPBEStringEncryptor.decrypt(password);
    System.out.println("username2:"+username2);
    System.out.println("password2:"+password2);
}

    下面是输出加密字符串：

    username:71gfTbIPb3/x04MWNzZ9Ww==
password:51A975NQFngJ9iAqvBwGYg==
username2:root
password2:root

  • 将加密后的字符串替换原明文

    application.yml

    spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/XXX
    username: ENC(71gfTbIPb3/x04MWNzZ9Ww==)
    password: ENC(51A975NQFngJ9iAqvBwGYg==)
    driver-class-name: com.mysql.jdbc.Driver

    注意: 上面的 ENC() 是固定写法.

  • 部署时配置salt(盐)值

    为了防止salt(盐)泄露,反解出密码.可以在项目部署的时候使用命令传入salt(盐)值:

    java -jar xxx.jar -Djasypt.encryptor.password=test_key

    这样就可以把配置文件里面的jasypt.encryptor.password删除了

或者在服务器的环境变量里配置。

打开/etc/profile文件

    vim /etc/profile

在profile文件末尾插入salt(盐)变量

    export JASYPT_PASSWORD = test_key

编译，使配置文件生效

    source /etc/profile

运行

    java -jar -Djasypt.encryptor.password=${JASYPT_PASSWORD} xxx.jar

备注

本文为使用自动配置类型的示例，@SpringBootApplication 或 @EnableAutoConfiguration

使用 Spring Boot 版本是 Version 2.1.X

使用 Jasypt 为 3.0.3 如果使用 Jasypt 版本为3.0以下，改方案不适用
